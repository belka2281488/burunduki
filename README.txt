ГАЛЕРЕЯ БУРУНДУКОВ КИРИЛА — ФОТО И ВИДЕО
===========================================

Эта версия хранит фото не на твоём компе, а в бесплатном облаке
Supabase. Благодаря этому сайт можно выложить на GitHub Pages,
и ЛЮБОЙ человек сможет заходить и добавлять своих бурундуков —
кнопка "Добавить бурундука" работает у всех, не только у тебя.

ШАГ 1. Создать бесплатный проект Supabase
-------------------------------------------
1. Зайди на https://supabase.com и зарегистрируйся (можно через GitHub).
2. Нажми "New Project".
3. Придумай имя проекта (например "burunduki") и пароль для базы
   (пароль просто сохрани куда-нибудь, он не понадобится в коде).
4. Подожди пару минут, пока проект создастся.

ШАГ 1.5. Создать таблицу в базе данных (для имён, описаний, редактирования)
------------------------------------------------------------------------------
1. В левом меню открой "SQL Editor".
2. Нажми "New query".
3. Открой файл setup.sql (лежит в этой же папке), скопируй всё
   содержимое и вставь в редактор.
4. Нажми "Run". Должно появиться "Success. No rows returned".

Это создаёт таблицу burunduki (имя, описание, ссылка на файл) и
таблицу burunduki_videos (ссылки на видео), и разрешает всем читать,
добавлять, редактировать и удалять записи в обеих.

ЕСЛИ ТЫ УЖЕ РАНЬШЕ ВЫПОЛНЯЛ setup.sql (до появления видео)
--------------------------------------------------------------
Не выполняй setup.sql заново — вместо этого выполни только
migration_add_videos.sql (он добавляет только новую таблицу для
видео, старые фото не тронет).

ЕСЛИ ТЫ УЖЕ РАНЬШЕ ВЫПОЛНЯЛ setup.sql (до появления оценок)
--------------------------------------------------------------
Не выполняй setup.sql заново — вместо этого выполни только
migration_add_ratings.sql (он добавляет только новую таблицу для
оценок, всё остальное не тронет).

СИСТЕМА ОЦЕНОК ("РАНГ БУРУНДУКА")
--------------------------------------
Вместо звёзд — ранг от 1 до 7 с картинкой:
1 sub3 — не красивый
2 sub5 — не особо красивый, но лучше
3 ltn — ближе к норм
4 mtn — красивенький
5 htn — красивый, крутой прям
6 chad — прям такой крутой очень
7 true burunduk — абсолютный пик красоты

Внутри лайтбокса фото и видео теперь есть панель оценки:
- сверху — средняя оценка (число от 1 до 7) и картинка ближайшего
  ранга, плюс сколько всего человек оценило;
- ниже — 7 кнопок с картинками, выбери одну и, по желанию, напиши
  комментарий, потом жми "Оценить";
- каждый человек (браузер) может поставить только одну оценку
  одному фото/видео — если оценить второй раз, старая оценка
  и комментарий просто обновятся;
- под кнопками — список всех оценок и комментариев от других людей.

Чтобы это заработало, нужно один раз выполнить migration_add_ratings.sql
(или, если ставишь сайт с нуля, достаточно обычного setup.sql —
там оценки уже включены).

ШАГ 2. Создать хранилище для фото (bucket)
---------------------------------------------
1. В левом меню открой раздел "Storage".
2. Нажми "New bucket".
3. Имя бакета: burunduki  (именно так, маленькими буквами)
4. Включи галочку "Public bucket" — иначе фото не будут открываться
   на сайте у других людей.
5. Создай.

ШАГ 3. Достать ключи для config.js
--------------------------------------
1. В левом меню зайди в "Project Settings" -> "API".
2. Скопируй "Project URL" — это твой SUPABASE_URL.
3. Скопируй ключ "anon public" — это твой SUPABASE_ANON_KEY.
4. Открой файл config.js в этом проекте и вставь оба значения
   вместо "ВСТАВЬ_СЮДА_...".

Эти ключи публичные, их можно смело класть в открытый репозиторий
на GitHub — они не дают доступ ко всей базе, только то, что
разрешено правилами бакета (мы включили публичный доступ на чтение
и запись для загрузки фото).

ШАГ 4. Открыть сайт локально и проверить
-------------------------------------------
Просто открой index.html в браузере (двойным кликом) — теперь
он читает фото прямо из Supabase, никаких выбора папки не нужно.
Нажми "Добавить бурундука", выбери фото, впиши имя (и, если хочешь,
описание) — оно появится в галерее сразу у всех, кто открывает сайт.

Внутри лайтбокса (когда открыл фото) теперь есть:
- ✏️ Изменить — поменять имя и описание
- 🗑️ Удалить — удалить бурундука насовсем (спросит подтверждение)

ШАГ 5. Выложить на GitHub Pages
------------------------------------
1. Создай новый репозиторий на GitHub (например "burunduki-gallery").
2. Залей туда все файлы из этой папки (index.html, style.css,
   script.js, config.js — ВМЕСТЕ с вставленными ключами).
3. В репозитории: Settings -> Pages -> Source -> выбери ветку main
   и папку / (root) -> Save.
4. Через минуту сайт будет доступен по ссылке вида
   https://твой-ник.github.io/burunduki-gallery/

Готово — теперь это настоящий сайт в интернете, доступный всем,
и любой сможет открыть его и добавить своего бурундука.

ПРО КНОПКУ "КОПИРОВАТЬ"
--------------------------
Она не работает при открытии файла напрямую (file://) —
это ограничение браузера. На GitHub Pages (там HTTPS) она
заработает сама, ничего доделывать не нужно.

КАК ДОБАВИТЬ ВИДЕО
----------------------
На сайте два раздела сверху: "🎬 Видео" и "🖼️ Фото".
В разделе видео жми "➕ Добавить видео", выбери источник:

- Google Drive — открой файл в Drive -> правой кнопкой -> "Настройки
  доступа" -> "Все, у кого есть ссылка" -> "Читатель" -> скопируй
  ссылку. Она выглядит так:
  https://drive.google.com/file/d/ДЛИННЫЙ_ID/view?usp=sharing
  Вставляй её как есть, сайт сам достанет нужную часть.

- YouTube — просто вставь обычную ссылку на видео или Shorts.

- TikTok — вставь ссылку на видео (кнопка "Поделиться" -> "Копировать
  ссылку"). Работает только если ссылка содержит числовой ID видео
  (обычно так и есть у прямых ссылок на видео).

- Tenor — вставь ссылку на гифку/видео.

- Другое — любая прямая ссылка на .mp4 или страницу с видео.

Видео открывается по клику на карточку, так же как и фото — с именем,
описанием, кнопками "Изменить" и "Удалить".

ПОИСК ПО ИМЕНИ
------------------
Сверху над галереей есть поле поиска — фильтрует карточки по имени
на лету, без перезагрузки страницы. Работает и для фото, и для видео.

ТОЛЬКО АВТОР МОЖЕТ РЕДАКТИРОВАТЬ СВОЁ
------------------------------------------
При первом заходе на сайт человек вводит своё имя (окошко "Привет!").
Оно сохраняется в его браузере вместе со случайным секретным кодом.
При добавлении фото/видео этот код записывается вместе с записью.

Кнопки "Изменить" и "Удалить" в лайтбоксе видны только если код
в браузере совпадает с кодом владельца записи — то есть только у
того, кто это добавил (на этом же устройстве и в этом же браузере).

ВАЖНО: это НЕ полноценная защита с паролями — если человек откроет
консоль браузера, он технически сможет подставить чужой код и
обойти это. Для галереи с друзьями обычно этого достаточно, но
если понадобится защита понадёжнее (настоящий вход с паролем) —
это отдельная, более сложная доработка через Supabase Auth.

Сменить своё имя можно кнопкой "сменить имя" в шапке сайта.

Старые фото/видео, загруженные до этого обновления, считаются
"общими" — их может редактировать кто угодно, потому что у них
нет привязанного владельца.

ЧТОБЫ SUPABASE НЕ "ЗАСЫПАЛ"
--------------------------------
Бесплатный проект Supabase автоматически замирает после 7 дней
без обращений — тогда сайт выдаст ошибку, пока кто-то не зайдёт
в панель Supabase и не "разбудит" проект вручную.

В архиве есть файл .github/workflows/keep-supabase-awake.yml —
он настраивает бесплатный робот на GitHub, который сам стучится
в базу раз в 3 дня, чтобы она не засыпала.

Чтобы он заработал, нужно один раз добавить 2 секрета в репозиторий:
1. На GitHub в репозитории: Settings -> Secrets and variables ->
   Actions -> New repository secret
2. Добавь секрет с именем SUPABASE_URL — значение как в config.js
3. Добавь секрет с именем SUPABASE_ANON_KEY — значение как в config.js
4. Готово — во вкладке "Actions" репозитория раз в 3 дня будет
   появляться зелёная галочка, значит пинг сработал.

Можно проверить сразу вручную: вкладка Actions -> Keep Supabase
awake -> Run workflow.

ЕСЛИ ЧТО-ТО ПОШЛО НЕ ТАК
----------------------------
- "Ошибка загрузки" на сайте -> проверь, что в config.js правильно
  вставлены URL и ключ, без лишних пробелов и кавычек по краям.
- Фото не отображаются у других -> проверь, что бакет "burunduki"
  создан именно как Public bucket (шаг 2).
- Хочешь ограничить, кто может добавлять фото -> в Supabase это
  настраивается через Authentication + Storage Policies, это уже
  следующий уровень, спроси отдельно, если понадобится.
